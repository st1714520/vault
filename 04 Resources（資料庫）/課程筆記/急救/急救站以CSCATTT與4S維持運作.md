# 急救站以 CSCATTT 與 4S 維持運作

災難急救站不是把醫療人員與器材集中在一個房間，而是把有限資源轉換成可以長時間運作、分流、處置及後送的系統。

## CSCATTT：從管理到醫療作業

### 管理層

- C — Command：建立單一指揮與角色責任。
- S — Safety：建物、感染、暴力、火災及人員安全。
- C — Communication：站內、醫院、指揮中心與後送單位的通訊。
- A — Assessment：傷患量、傷型、資源、道路及醫院量能評估。

### 作業層

- T — Triage：依緊急程度及可用資源檢傷分類。
- T — Treatment：執行穩定、急救、留觀與持續重評。
- T — Transport：安排後送、逆後送及交班。

## 4S：急救站能否持續運作

| 要素 | 核心問題 |
|------|----------|
| Staff | 誰值班？角色、資格、排班、休息與替補是否清楚？ |
| Stuff | 醫材、藥品、PPE、水、電、照明、通訊與耗材能撐多久？ |
| Space & Structure | 檢傷、候診、治療、留觀、清消、後送與人員休息是否分區？ |
| System | 如何啟動、紀錄、補給、轉診、感染控制與結束任務？ |

## 空間與動線

- 入口先檢傷與必要清消。
- 候診、立即處置、留觀、出院及後送區分開。
- 傷患移動盡量單向，避免與物資、人員休息及污染動線交叉。
- 保留隱私、手部衛生、廢棄物與罹難者管理空間。
- 規劃備援電力、用水、通訊及救護車進出。

## 急救站不是醫院的縮小版

急救站應聚焦於檢傷、初步穩定、基本照護、短期留觀與雙向轉診。超出人力、設備或持續照護能力的傷患，應儘早與責任醫院協調，而不是在站內勉強完成高風險處置。
