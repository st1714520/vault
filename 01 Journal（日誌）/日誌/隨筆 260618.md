AI能整理知識，但還不能替我思考

今天花了不少時間整理 Obsidian，並成功將它與 Claude Code 串接起來。過去我的筆記散落在不同地方，許多想法雖然曾經深入思考過，但隨著時間流逝，很難再次找到。如今終於有機會把這些內容慢慢收攏到同一個系統裡。

在整理過程中，我開始測試 Claude Code 是否能協助建立卡片盒式的知識庫。我讓它分析自己的文章與讀書心得，並嘗試拆分成較小的筆記。

最初的體驗其實有些失望。

我原本期待的是，它能從大量筆記中發現我沒有意識到的深層結構，甚至找出跨主題的共同問題意識。然而實際上，它更擅長的是保留原本內容、重新命名標題、補上標籤，以及建立一些表面合理的關聯。

例如幾篇從佛洛姆與費登奎斯讀書心得拆出的筆記，內容基本上都保留了原本的意思，標題也大多抓到重點。但當它開始建立關聯時，錯誤率便明顯提高。許多連結看似合理，實際上只是因為出現了相同關鍵字，而非真的討論同一個問題。

我也發現另一個現象。

當我要求它拆解心得時，它經常傾向把整段心得保留在同一張卡片裡，而不是抽出其中值得獨立存在的觀察。換句話說，它比較像是在幫我整理資料，而不是幫我形成永久筆記。

經過這次測試後，我開始重新理解 AI 在知識管理中的角色。

目前看來，它比較像是一位圖書館員：

- 幫忙命名
- 幫忙分類
- 幫忙搜尋
- 幫忙摘要

但當涉及真正重要的事情，例如：

- 什麼值得成為一張永久筆記？
- 哪些概念其實在談同一個問題？
- 哪些文章背後有共同的張力？

它仍然經常停留在主題層，而不是結構層。

有趣的是，當我請它分析人格時，雖然許多推論過於籠統，但整體方向卻不算離譜；反而在我以為較容易的知識拆解工作上，暴露出更多問題。

或許這正好說明了一件事：

AI 很擅長保留我的原意，卻還不擅長理解我為什麼會這樣想。

至少目前如此。

不過即便如此，我仍然對這套系統感到期待。因為真正讓我興奮的，並不是 AI 幫我下結論，而是終於有能力把多年累積的夢境、讀書心得、創作想法與觀察放進同一個容器裡。未來就算 AI 無法替我思考，它至少能幫我找到那些曾經思考過的東西。

而這或許已經足夠有價值了。
